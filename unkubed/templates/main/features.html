Feature overview

Pages that map UI to kubectl

These screens focus on showing Kubernetes resources, the commands behind them, and the supporting YAML or troubleshooting context.

Contextual dashboard

A single glance shows namespaces, workloads, services, and the exact kubectl get commands that populate them.

Template builder

Generate deployment, service, and config map YAML, preview the manifest, and apply it from the UI when a cluster is connected.

Pod troubleshooting

Combine events, logs, and restart counts into a short troubleshooting summary with suggested next checks.

Command history

Stores the commands triggered through the interface along with their status, output, and timestamps.

Cluster connections

Lets the user save kubeconfig and context combinations and switch the active connection from the UI.

Saved templates

Keeps generated manifests in the database so they can be revisited and reused later.
